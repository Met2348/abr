## 1. 项目定位与目标

**核心任务**：构建 Process Reward Model（PRM），用于数学推理多步骤验证。

**模型架构**：冻结 Qwen2.5-Math-PRM-7B 骨架 + 可训练 MLP 价值头（"BCR/ABR 控制器"的评分器组件）。

**主要指标**：ProcessBench F1（官方定义）= 2×Acc_error×Acc_correct / (Acc_error + Acc_correct)。
- Acc_error：模型正确识别含错误步骤链的比例
- Acc_correct：模型正确识别无错误链的比例
- 决策规则：扫描序列中首个 score < τ 的步骤，若存在则判为错误链

**目标**：MATH F1 > 0.735（社区 Qwen2.5-Math-PRM-7B full = ~0.735）

## 4. 反模式诊断（Anti-Pattern A–G）

### 4.1 反模式 A：几何/长度偏差（最严重）

**发现**：Math-Shepherd 的 bad_prefix 平均比 good_prefix **长 194 tokens**。
**影响**：模型在 ProcessBench 上反转评分——长步骤链被判为"坏"，短的被判为"好"。
**修复**：使用 DPO 数据（从正确解答中截取合法 prefix）。
**验证**：FIX_C（纯 DPO 8k）= MATH F1 0.721（证明 DPO 数据几何正确）。

### 4.2 反模式 B：Terminal 比例失衡

**发现**：terminal_top1 ≈ 0.05 时，40–48% 全正确样本无监督，terminal BCE 无法学习。
**修复**：降低 min_conf=0.55，增加 terminal pair 覆盖；lambda_terminal_bce=0.25。

### 4.3 反模式 C–G

- C（置信度）：min_conf 从 0.70 放宽至 0.55，增加覆盖
- D（F1 指标）：ProcessBench F1 正确实现 ✅
- E（dual_head 路由）：代码实现 ✅，但实验显示 dual_head 在主线数据下不稳定
- F（R-PRM 格式）：compact_verdict 确认正确 ✅
- G（混合数据危险组合）：recipe_safety.py 封锁已知致命组合

## 6. Phase E LoRA 系列：第二轮（PBR33–PBR37, L1）

### 6.1 实验设计

第二轮目标：通过系统消融确认 PBR32 结果的泛化性，尝试进一步提升。

实验矩阵：
- PBR33：r=8 top-4, PBR26 数据（vs PBR32 all-28/PBR12）
- PBR34：r=16 top-4, PBR26 数据（vs PBR33 高 rank）
- PBR35：r=8 all-28, PBR26 数据（vs PBR32 换数据）
- PBR36：r=32 all-28, PBR26 数据（vs PBR35 更高 rank）
- PBR37：r=8 all-28 + contrastive=0.2, PBR26 数据（vs PBR35 + 对比损失）
- L1：r=8 all-28 + contrastive=0.05 + centering=0.01, PBR26 数据

### 6.2 全部结果（已确认）

| Run | MATH F1 | GSM F1 | vs PBR32 | 配置要点 |
|-----|---------|--------|----------|---------|
| **PBR32 ★** | **0.689** | 0.776 | baseline | r=8 all-28, PBR12 |
| PBR33 | 0.666 | **0.797** | -0.023 | r=8 top-4, PBR26 |
| PBR34 | 0.657 | 0.766 | -0.032 | r=16 top-4, PBR26 |
| PBR35 | 0.657 | 0.768 | -0.032 | r=8 all-28, PBR26 |
| PBR36 | 0.656 | 0.739 | -0.033 | r=32 all-28, PBR26 (ep1) |
| PBR37 | **0.655** | 0.769 | -0.034 | r=8 all-28 + ctr=0.2, PBR26 |
| L1 | 0.654 | 0.762 | -0.035 | r=8 all-28 + ctr=0.05 + center, PBR26 |

### 6.3 关键发现：三个"无关性定理"

**定理 1：LoRA rank 无关性**
r=8 → r=16 → r=32 的 MATH F1：0.657 → 0.657 → 0.656（无显著差异）。
可训练参数：2.52M → 5.06M → 10.1M，增加 4× 不带来任何提升。
**结论**：信息瓶颈不在模型容量，在训练信号质量。

**定理 2：LoRA 层数无关性**
top-4 层 vs all-28 层的 MATH F1：0.657 vs 0.657（完全相同，使用 PBR26 数据时）。
**结论**：深层特征表示对 ProcessBench 迁移无额外贡献（PBR26 数据条件下）。

**定理 3：对比损失有害**
PBR35（无对比损失）= 0.657 > PBR37（ctr=0.2）= 0.655 > L1（ctr=0.05）= 0.654。
对比损失权重越大，MATH F1 越低。
**推测**：对比损失推动 chosen/rejected 特征分离，破坏了 ProcessBench 所需的分布外泛化能力。

### 6.4 决定性因素：数据质量

PBR35（PBR26 数据, r=8 all-28）= 0.657 vs PBR32（PBR12 数据, r=8 all-28）= 0.689：**差距 0.032**

数据差异：
| 数据集 | 训练对数 | MS 来源 | MATH F1（LoRA r=8） |
|--------|---------|---------|-------------------|
| PBR12 | 5,705 | DPO + MS strict（~3,307 对） | **0.689** |
| PBR26 | 7,366 | DPO + MS full（~4,968 对） | 0.657 |

PBR26 的 MS full 数据含更多噪声（MC 估计置信度较低）。LoRA 放大了训练信号，因此噪声也被放大。**冻结骨架充当了天然正则化器**——它的 inductive bias 过滤了噪声，但也限制了上限。

**反直觉推论**：删除 1,661 个噪声训练对（PBR26→PBR12），MATH F1 提升 0.032。

### 7.2 各实验假设

**PBR38（termBCE=0.5）**：
假设：当前 termBCE=0.25 对 terminal 步骤锚定不足。加倍到 0.5 应使 terminal 步骤预测更准确，提升 acc_erroneous。
风险：过强的 terminal 监督可能破坏局部步骤分辨率。

**PBR39（lr=1.5e-5）**：
假设：lr=3e-5 可能导致 LoRA 过拟合 PBR12 训练分布。降低 LR（0.5×）提供更保守的骨架更新，可能改善泛化。
期望：MATH F1 ≈ 0.688-0.692（谨慎乐观）。

**PBR40（anti_sat=5e-3）**：
假设：logit 饱和（分值趋于 0/1 极端）可能导致 acc_erroneous 偏低，因为模型对"不确定"步骤也给出极端预测。更强的抗饱和正则化使分值分布更平坦，可能改善阈值分辨率。
风险：过强正则化可能降低 pair_acc。

### 7.3 预期结果

基于第二轮消融的规律，第三轮可能的结果范围：
- 乐观：MATH F1 = 0.690–0.695（+0.001–+0.006 vs PBR32）
- 悲观：MATH F1 = 0.684–0.689（持平或略低）
- 失败：MATH F1 < 0.686（低于冻结上限）

完成时间预估：~3-4 小时后（每个实验约 5 个 epoch × ~30 分钟/epoch）。

### 9.3 Clip+Delta 奖励整形

**已实现**：`PRMScorer.score_clip_delta()` in `scripts/phase_f_grpo_lite.py`。

## 10. 关键技术洞察与因果图

### 10.1 "数据质量 > 模型复杂度"原则

```
PBR12（5705 对, strict MS）    ─── LoRA r=8 all-28 ──→  MATH F1 = 0.689  ★
PBR26（7366 对, full MS ）    ─── LoRA r=8 all-28 ──→  MATH F1 = 0.657
PBR26（7366 对, full MS ）    ─── 冻结 MLP         ──→  MATH F1 = 0.686
```

直觉：LoRA 微调放大训练信号——好信号 → 好特征，坏信号 → 过拟合分布偏差。

### 10.2 acc_erroneous 是主要瓶颈

| 模型 | acc_erroneous | acc_correct | MATH F1 |
|------|-------------|-------------|---------|
| 冻结 PBR26 | 0.572 | 0.856 | 0.686 |
| LoRA PBR32 | 0.564 | 0.884 | 0.689 |
| 社区 EDU-PRM | ~0.82+ | ~0.95+ | 0.882 |

**突破路径**：提升 acc_erroneous 0.57→0.82 可使 F1 从 0.689→0.88+。
**实现方式**：更准确的错误步骤标注（EDU-PRM 熵边界），或更多高质量错误链训练对。

### 10.3 F1 公式的临界性质

F1 = 2 × 0.564 × 0.884 / (0.564 + 0.884) = **0.689**
对称改善：若 acc_err +0.01 且 acc_cor +0.01 → F1 ≈ +0.010
非对称：提升 acc_err +0.02（0.584→0.606）→ F1 ≈ 0.700+
